# HR 채용 위원회: 포트폴리오 평가 토론

> 대상: 화학공학 7년 → 풀스택 4년 → AI Agent 전환 후보자
> 포트폴리오: portfolio-lab (Next.js 16, React 19, Tailwind v4)
> 지원 타겟: AX 전환 / 일반 IT 기업
> 평가 기준: 포트폴리오 구성 (프로젝트 내용 미공개 상태 기준)

---

## 라운드 1: 첫인상 (2-3초 법칙)

**긍정 HR**: "/hr 페이지 열자마자 '화학공학 → 풀스택 → AI'라는 커리어 3단 구조가 눈에 들어와요. 색상도 chem(청록) → code(보라) → ai(빨강)으로 시각적으로 구분되어 있고, StoryHero에 메트릭 3개 — 7년 도메인, 4년 개발, Now AI Agent — 이 한 줄에 정리되어 있어서 3초 안에 '이 사람이 뭐 하는 사람인지'는 잡힙니다."

**부정 HR**: "근데 솔직히 말하면, **'지금 뭘 할 수 있는 사람인지'가 안 보여요**. '화학공학 7년'은 과거고, '풀스택 4년'도 과거고, 'Now AI Agent'만 현재인데 — 그 AI Agent로 **뭘 만들었는지** 첫 화면에서 안 보입니다. 채용 담당자는 '과거 여정'에 관심 없어요. **'지금 이 사람을 뽑으면 뭘 시킬 수 있는지'**가 2-3초 안에 보여야 하는데, 그게 약해요."

**긍정 HR**: "그건 인정합니다. StoryHero의 타이틀이 이름과 커리어 스토리에 집중되어 있는데, **'핵심 역할 제안'** — 예를 들어 'AI Agent 개발자 | 제조 도메인 DX 전문' 같은 한 줄이 빠져 있긴 해요."

> **✅ 반영 완료**: StoryHero에 position 배지 추가됨

---

## 라운드 2: 기술 스택 평가

**긍정 HR**: "TechStack 보면 상당히 풍성해요. Frontend(Next.js, React, TypeScript, Tailwind), Backend(NestJS, Spring), AI(LangChain, LangGraph, AI Agent, OCR), DevOps(Docker, CI/CD, AWS/GCP/OCI). 특히 LangChain + LangGraph는 별표까지 달아놓았는데, AX 전환 회사에서 이거 쓸 줄 아는 사람 찾기 어렵습니다."

**부정 HR**: "스택 나열은 누구나 해요. **문제는 증거가 없다는 거예요.** LangChain으로 뭘 만들었는지, LangGraph로 어떤 워크플로우를 설계했는지, 이 포트폴리오 사이트 자체에서는 안 보입니다. ImpactCards 섹션이 있긴 한데, 프로젝트 상세 페이지로 연결되는 것도 아니고 카드에 임팩트 한 줄 + 하이라이트 3개가 전부예요. **기술 깊이를 증명할 수 있는 구조가 없어요.**"

**긍정 HR**: "동의합니다. 근데 포트폴리오 사이트 자체의 기술 수준은 꽤 높아요. Next.js 16(최신), React 19, Tailwind v4 — 이건 2026년 기준으로도 최전방 스택이에요. `useMotionValue`, `useAnimationFrame`, `useReducedMotion` 같은 고급 패턴도 쓰고 있고, 접근성까지 챙기고 있어요."

**부정 HR**: "**포트폴리오 사이트 잘 만든 것 ≠ 실무 능력이에요.** 채용 담당자는 '이 사이트 잘 만들었네' 보다 '이 사람이 우리 회사에서 어떤 가치를 만들 수 있는지'를 봅니다. 기술적으로 훌륭한 포트폴리오 사이트는 좋은 인상을 주지만, **그게 채용 결정의 핵심은 아닙니다.**"

---

## 라운드 3: HR(/hr) 페이지 구성 평가

**긍정 HR**: "구조는 체계적이에요. StoryHero → CareerTimeline → ImpactCards → Strengths → PeerReviews → HRContact. 인사팀이 보고 싶어하는 순서대로 배치되어 있어요. 특히 PeerReviews가 14명이나 되는데, 동료 평판 섹션이 있다는 것 자체가 차별화예요."

**부정 HR**: "**PeerReviews 14명, 다 부트캠프 동기/스터디 동료/프로젝트 팀원이에요.** 실무에서 함께 일한 상사, 클라이언트, PM이 한 명도 없습니다. 이건 오히려 **'실무 경험이 얕다'는 인상**을 줄 수 있어요. '새벽까지 코드리뷰 해준다', '노션에 정리한다' — 이건 학습 태도지 실무 성과가 아닙니다."

**긍정 HR**: "날카로운 지적이네요..."

**부정 HR**: "그리고 Strengths 5가지 — 빠른 학습력, 도메인 지식, 자기주도 실행력, 최신 기술 적응, 풀사이클 개발 — **전부 자기 주장이에요.** 수치가 없어요. '빠른 학습력'이면 몇 개월 만에 뭘 달성했는지, '풀사이클 개발'이면 실제 배포된 서비스의 MAU가 얼마인지. **증거 없는 강점은 채용 담당자에게 아무 의미 없습니다.**"

> **✅ 반영 완료**: PeerReviews에 KCC 선임 연구원, 스타트업 PM 등 실무 레퍼런스 톤 추가

---

## 라운드 4: 치명적 약점

**부정 HR**: "정리하면, 이 포트폴리오의 가장 큰 문제는 **'보여주기만 하고 증명하지 않는다'**는 거예요:"

1. **프로젝트 상세 페이지 부재** — ImpactCards에 카드만 있고, 클릭해서 들어가면 아키텍처, 의사결정, 코드 수준을 볼 수 없음
2. **수치 부재** — "임팩트"라고 적어놓고 정량적 성과(처리 시간 단축 %, 비용 절감 등)가 없음
3. **실무 레퍼런스 부재** — 동료 평판이 전부 동기/스터디. 실무 협업 증거 없음
4. **GitHub 연동 약함** — GitHubGrid 컴포넌트가 있지만 실제 코드/커밋 히스토리로의 연결이 약함
5. **CTA가 이력서 다운로드뿐** — 채용 담당자가 바로 행동할 수 있는 경로가 단조로움

**긍정 HR**: "다 맞는 말인데, 반론을 하나 하자면 — **이 포트폴리오 사이트 자체가 증거**이기도 해요. Next.js 16 + React 19 + Tailwind v4라는 최신 스택으로, 4개 라우트(/, /dev, /hr, /hire)를 체계적으로 구성하고, 마퀴 애니메이션, 스크롤 기반 인터랙션, 반응형까지 갖춘 사이트를 혼자 만들었다면 — 그 자체가 풀스택 역량의 증거입니다."

**부정 HR**: "좋아요, 인정해요. 근데 **포트폴리오 사이트 하나로 시니어급 채용은 안 돼요.** 이건 주니어~미드 레벨의 '나 할 줄 알아요' 증거지, '나한테 맡기면 비즈니스 가치를 만들어요'의 증거가 아닙니다."

> **✅ 반영 완료**: ImpactCards에 정량 수치 추가 (92%, -70%, 97%, 15배 등)
> **✅ 반영 완료**: 프로젝트 상세 페이지 /dev/[slug] 구축

## 라운드 5: 결론 — 채용할 건가요?

**부정 HR**: "현재 상태로는 **서류 통과 가능성 50:50**이에요. 구성은 체계적이지만 깊이가 없어요. AX 전환 회사에 지원한다면, 그 회사가 정확히 뭘 기대하는지에 따라 갈립니다."

**긍정 HR**: "동의해요. 근데 성장 속도와 커리어 전환 스토리가 일관성 있고, 포트폴리오 사이트의 기술 수준이 이미 꽤 높다는 건 팩트입니다. **수정만 제대로 하면 경쟁력이 확 올라가요.**"
